# Contributing

Contributions are welcome! Please see our guidelines for:
- Adding new scenarios
- Improving documentation
- Reporting issues
- Submitting pull requests

## Process

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/scenario-x`)
3. Commit your changes (`git commit -am 'Add new scenario'`)
4. Push to the branch (`git push origin feature/scenario-x`)
5. Submit a Pull Request

## Code Style

- Use 4 spaces for indentation
- Follow PEP 8 for Python code
- Include docstrings for all functions
- Add comments for complex logic

## Testing

Before submitting:
- Run `pytest` to verify tests pass
- Test scenarios in both mock and real modes
- Update documentation if needed

Thank you for contributing!
